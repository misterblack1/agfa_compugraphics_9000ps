# AGFA-MON: Custom ROM Firmware for Agfa Compugraphic 9000PS

This directory contains the source and build artifacts for **AGFA-MON**, the custom replacement firmware for the Agfa Compugraphic 9000PS main processor board.

## What is AGFA-MON?

**`agfa-mon.m68`** is the primary ROM source code (68020 assembly language). It contains:
- The AGFA-MON machine language monitor - interactive command-line interface for hardware control
- EhBASIC 3.54 interpreter - embedded directly in the ROM
- Boot sequence and initialization code
- Hardware interface routines

## Building and Installing

### Source to Board: 4-Part ROM Splitting

The Agfa 9000PS has **4 ROM chips** on the main board. AGFA-MON cannot be installed as a single binary - it must be split into 4 parts matching the hardware layout.

The build process:
1. Generate `include/build_version.inc` with current build timestamp
2. Assemble `agfa-mon.m68` using the vasm assembler → `agfa-rom.bin` (128KB flat binary)
3. Split the flat binary into 4 parts matching the hardware layout
4. This produces **4 binary files:**
   - `agfa-rom-LL.bin` - Low address, Low bank
   - `agfa-rom-LM.bin` - Low address, Mid bank
   - `agfa-rom-HM.bin` - High address, Mid bank
   - `agfa-rom-HH.bin` - High address, High bank

**These 4 files are what you physically install into the ROM sockets on the board.**

### Building the ROM

#### Option A: PowerShell (Windows 11 / All Platforms with PowerShell)

From the `agfa-monitor/` directory:
```powershell
.\build.ps1          # Build everything (assemble + split)
.\build.ps1 -Clean   # Remove generated files
.\build.ps1 -Help    # Show help
```

**Requirements:**
- Python 3 installed and in PATH
- vasm assembler at `../assembler/vasmm68k_mot.exe`

#### Option B: GNU Make (Linux/Mac/Unix or Windows with Make installed)

From the `agfa-monitor/` directory or project root:
```bash
make              # Build all (assembly + split)
make chips        # Same as 'make all'
make clean        # Remove generated files
make help         # Show help
```

If you don't have GNU Make on Windows, use the PowerShell script (Option A) instead.

### No Simulator

There is no software simulator for the Agfa 9000PS. All testing and debugging must be done on actual hardware.

## Building AGFA-MON

### Quick Start

1. Edit `agfa-mon.m68` as needed
2. From the `agfa-monitor/` directory, run:
   - **PowerShell:** `.\build.ps1`
   - **GNU Make:** `make`
3. This produces the 4 ROM chip files (agfa-rom-HH.bin, agfa-rom-HM.bin, agfa-rom-LM.bin, agfa-rom-LL.bin)
4. Use a EEPROM programmer (TL866-3G in AM27C256 mode) to flash the 4 ROM chips
5. Install the 4 chips back into the board

See **"Building the ROM"** section above for detailed build instructions.

### Build Version String

Both build scripts automatically generate a build version string with the current date and time. This string is included in the ROM banner displayed on power-up:

```
Agfa Compugraphic 9000PS
by Adrian Black / Adrian's Digital Basement 2026

BUILD 06-APR-2026 14:30
...
```

The version is auto-generated in `include/build_version.inc` each time you build, so you always know when the ROM was compiled.

## EhBASIC Integration

EhBASIC 3.54 is embedded directly into AGFA-MON. The BASIC interpreter runs as part of the main firmware, accessible via AGFA-MON commands (type `B` at the monitor prompt to enter BASIC).

The BASIC source and include files are here for reference and potential modification. Changes to EhBASIC require rebuilding the entire ROM.

## Modifying AGFA-MON

When making changes:
1. Edit `agfa-mon.m68`
2. Follow the build process above
3. Test on actual hardware (no simulator available)
4. Use AGFA-MON's debugging commands to inspect hardware state

Common modifications:
- Adding new monitor commands
- Changing boot behavior
- Extending hardware interfaces
- Modifying or extending EhBASIC
